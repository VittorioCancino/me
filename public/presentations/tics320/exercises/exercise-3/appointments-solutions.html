clients(client_id, client_name, phone)

staff(staff_id, staff_name)
appointments
appointments(client_id, appointment_date, appointment_time, service, staff_id)
Solution

Appointment time and client name for Giuliano on Feb 14.

Reduced RA
\[ \pi_{Clients.name,\;Appointments.time} \left( \left( \pi_{CID, SID, time} \left( \sigma_{date = \text{'Feb14'}}(Appointments) \right) \right) \;\bowtie_{Appointments.SID = Staff.SID}\; \left( \pi_{SID} \left( \sigma_{name = \text{'Giuliano'}}(Staff) \right) \right) \;\bowtie_{Appointments.CID = Clients.CID}\; Clients \right) \]
Solution

Appointment time and client name for Giuliano on Feb 14.

Step By Step
\[ R_1 = \sigma_{date = \text{'Feb14'}}(Appointments) \] \[ R_2 = \pi_{CID, SID, time}(R_1) \] \[ R_3 = \sigma_{name = \text{'Giuliano'}}(Staff) \] \[ R_4 = \pi_{SID}(R_3) \] \[ R_5 = R_2 \;\bowtie_{R_2.SID = R_4.SID}\; R_4 \] \[ R_6 = R_5 \;\bowtie_{R_5.CID = Clients.CID}\; Clients \] \[ \text{Result} = \pi_{Clients.name,\;R_5.time}(R_6) \]
Solution

Same query, but begin with all three relations in full.

Reduced RA
\[ \pi_{Clients.name,\;Appointments.time} \left( \sigma_{ Appointments.date = \text{'Feb14'} \;\wedge\; Staff.name = \text{'Giuliano'} \;\wedge\; Appointments.SID = Staff.SID \;\wedge\; Appointments.CID = Clients.CID } \left( Appointments \times Staff \times Clients \right) \right) \]
Solution

Same query, but begin with all three relations in full.

Step By Step
\[ R_1 = Appointments \times Staff \] \[ R_2 = R_1 \times Clients \] \[ R_3 = \sigma_{ Appointments.date = \text{'Feb14'} \;\wedge\; Staff.name = \text{'Giuliano'} \;\wedge\; Appointments.SID = Staff.SID \;\wedge\; Appointments.CID = Clients.CID }(R_2) \] \[ \text{Result} = \pi_{Clients.name,\;Appointments.time}(R_3) \]
Solution

Which answer is better?

Answer

The first query is better because it reduces the size of intermediate results by applying selection and projection early.

The second query builds a large Cartesian product before filtering, which makes it less efficient.

In relational algebra, both queries are equivalent in result, but in practice the first approach is preferred because it leads to better execution performance.
