Database Context

The dvdrental Database

This PostgreSQL tutorial database contains 15 relations in total.

For this advanced SQL exercise block, we still do not need the full database. We will expand the basic working subset so we can practice grouping, aggregation, filtering groups, and richer multi-relation queries.

For now, we will work with 10 relations.

Source database: Neon dvdrental sample

Working Subset

The Relations We Will Use

First, we keep the core relations from the earlier SQL exercises.

customer(customer_id, store_id, first_name, last_name, email, address_id, activebool, create_date, last_update, active)

inventory(inventory_id, film_id, store_id, last_update)
rental(rental_id, rental_date, inventory_id, customer_id, return_date, staff_id, last_update)

Questions · Part 1

First Aggregate Queries

  1. Query 1

    Count how many customers are in the `customer` relation.

  2. Query 2

    Count how many films are in the `film` relation.

  3. Query 3

    Show the average rental rate of all films.

  4. Query 4

    Show the maximum payment amount in the `payment` relation.

i Hint

Start with `COUNT`, `AVG`, `MAX`, and a single relation before combining grouping or joins.

Questions · Part 2

Aggregates with Simple Filters

  1. Query 5

    Count how many films have rating `PG`.

  2. Query 6

    Show the average length of films with rating `PG-13`.

  3. Query 7

    Show the total amount paid by the customer with `customer_id = 1`.

  4. Query 8

    Count how many rentals were returned (`return_date` is not null).

i Hint

Read these as: first choose the rows with `WHERE`, then apply the aggregate over the remaining rows.

Questions · Part 3
